# Требования к системе
В системе должен быть установлен **Docker** и **Containerlab**.
В качестве узлов сети используются *Docker-контейнеры*, для которых нужно собрать образы.
Сделать это можно с помощью цели *make*:

```bash
make images
```

# Запуск лабораторной работы
## Выбор лабораторной работы

Для выбора работы используется *make* c передачей номера работы в качестве переменной окружения `LAB`.  
Например, выбор первой работы:

```bash
make deploy LAB=lab1
```

## Остановка стенда
Для остановки стенда и удаления всех контейнеров используется цель для *make*:

```bash
make destroy
```

## Статус стенда
Для получения статуса стенда используется цель для *make*:

```bash
make inspect
```

# Подключение к хостам стенда
Для подключения можно использовать **ssh**. Для этого подготовлены алиасы, их нужно только активировать:

```bash
source scripts/activate
```

После этого в сессии терминала будут доступны команды для подключения к устройствам:

```bash
spine1
spine2
leaf1
leaf2
leaf3
```

Для убирания алиасов добавлена отдельная команда:

```bash
deactivate
```

# Схема стенда
Для графической демонстрации стенда можно использовать *Visual Studio Code* и расширение *Containerlab*.
Средство просмотра топологии открывается по сочетанию клавиш: *Ctrl + Shift + P*

# Wireshark
Для трассировки пакетов можно использовать **wireshark**. Он поддерживает работу с потоком байт, от **tcpdump**. 
Сделать это можно с помощью команды:

## Ssh
```bash
wireshark -k -i <(sshpass -p root ssh root@clab-lab-spine1 tcpdump -U -nni eth1 -w -)
```

## Netns
```bash
wireshark -k -i <(sudo ip netns exec clab-lab-spine1 tcpdump -U -nni eth1 -w -)
```

Предварительно нужно выполнить какую-нибудь команду от **sudo**, чтобы избежать запроса пароля. 

Эта команда запустит *tcpdump* в сетевом пространстве контейнера **spine1** на интерфейсе *eth1*.
